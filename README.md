# C++ for Java Developers

> A practical guide for Java developers who want to solve DSA and LeetCode problems in modern C++.

This repository is **not** a complete C++ course.

It is a focused set of notes that helps bridge the gap between Java and modern C++ while keeping the learning process practical.

The goal isn't to master every feature of C++.

The goal is to become comfortable writing algorithms in C++ without spending months learning parts of the language that are irrelevant to DSA.

---

## Why this repository exists

As a Java backend developer, I wanted to solve DSA and LeetCode problems in C++ because of its excellent Standard Template Library (STL) and its popularity in competitive programming.

However, I didn't want to:

- Read another 1000-page C++ book
- Watch dozens of hours of tutorials
- Learn every corner of the language

Instead, I decided to learn C++ organically by translating problems I had already solved in Java.

These notes are the result.

---

## Who is this repository for?

This repository is ideal if you:

- Know Java
- Have basic programming knowledge
- Want to solve LeetCode in C++
- Want to understand the STL
- Prefer learning by comparison instead of starting from scratch

It is **not** intended for learning advanced C++ development.

## Repository Philosophy

These notes intentionally avoid documenting the entire language.

Instead, they focus on concepts that are repeatedly useful while solving DSA problems.

The idea is simple:

> Learn only what helps you write better algorithms.

Everything else can be learned when it becomes necessary.
